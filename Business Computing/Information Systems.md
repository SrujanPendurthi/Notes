Developing an information system is based on the 4-step process
The first three are Systems Analysis
	Define and understand the problem - Requirements Analysis
	Develop Alternative Solutions - May range from low to high cost
	Choose the best solutions - feasibility study and Systems proposal report
	Implement the solution - Systems Design, Testing, Training

Traditional Systems Development Lifestyle
	Used for building large, complex systems
	Mostly a serial process, rigorous and formal
	Drawbacks: Very time-consuming and not flexible

Prototyping Method
	Built rapidly & inexpensively
	Very useful in discovering requirements
	Especially useful in designing a user interface
	Drawbacks: 

Spiral Model
	Evolutionary Software Process Model
	Iteration and sequential characteristics
	1st pass: planning/product specification
	2nd pass: Prototype/initial implementation
	3rd pass: revised implementation

Joint-Application Design
	Bring all departments/groups with leaders, users, analysts together in order to resolve conflict and obtain consensus
	Drawbacks: Expensive, Airtime, and Social Pressure

Rapid Application Development
Agile Methods
	Break Projects into small-sub projects
	Continual Customer Involvement
	Iteration & Release Plans

End-User Development
	End Users create simple information systems with little or no assistance from technical specialists
	Use user-friendly query, reporting, graphics, Web site development, and PC software tools to develop information systems
	Completed more rapidly than systems developed with conventional tools

Purchasing Solutions: Application Software Packages
	Determine User functional and system performance requirements
	Prepare Request for Proposal
	Application software packages that are generalized systems for universal functions with standard processes with varying levels of customization
	Outsourcing(Domestic vs Offshore)

Outsourcing
	Benefits:
		Cheaper
		Flexibility: No need to hire more employees
		May have deep technical expertise
		Non-Strategic assets
	Risks:
		Underestimating language and cultural barriers
		Lack of legal knowledge
		High Employee turnover at Vendor Company
		Hidden costs

Mobile Application Development
	Mobiles Web Sites
	Mobile Web App: internet enabled app accessed by web browser
	Native App
	Different Requirements for mobile devies than for PCs
		Reduced screen sizes and touch screens
		Saving resources
	Responsive Web Design

Project Mangement
Application of knowledge, skills, tools, and techniques to achieve targest within specified budget and time constraints
Five major variables
	Scope
	Time
	Cost
	Quality - how well the result satisifies the objectives
	Risk - problems that may threaten the success of project

Feasibilty Study
	Requirements - can the team implement the user functional and system performance requirments
	Technical - does the project team have the technological expertise
	Operational 
	Economic

Stakeholder Analysis
	Groups that can affect or will be affected by the projects
	Step 1: identify the major stakeholders for the projects
	Step 2: identify the critical assumptions and expectations of the major stakeholders for the project
	Common stakeholders - team, end users, and company

Major Drives of IT Risks
	External Factors
	Relative Project Size
	Requirements Volatility - changes in organization, procedures and support
	Technology Familiarity - low technical expertise

Project Scoping
Project Approval - only give funding after approval
Project Organization - define responsibilities of Project Team
Project Management Software - tracks progress and coordinates tasks
Milestone Reviews - detect errors, reduce network and delays


Selecting Projects:
	Determining Project Costs and Benefits
		Tangible(Measured in Dollars) Benefits vs Intangible
		Net Benefits
		Capital Budgeting Methods(NPV, IRR)
	Information Systems Plan: How well does the system support the IS plan?
	Portfolio Analysis:
		Scoring Model: Score each alternative based on chosen criteria and weights
